Bellman-Ford算法的实现与优化

Posted on 2020-01-15 Edited on 2020-01-22

题目来源：Leetcode-743.Network Delay Time

题目描述：There are N network nodes, labelled 1 to N.

Given times, a list of travel times as directed edges times[i] = (u, v, w), where u is the source node, v is the target node, and w is the time it takes for a signal to travel from source to target.

Now, we send a signal from a certain node K. How long will it take for all nodes to receive the signal? If it is impossible, return -1.

题目大意：给定一个有向图，求某一点到各个点最短路径中的最大值。若该点到其他某一点不可达，则返回-1。

本题可以使用很多方法，比如说Dijskra或者Floyd算法。这里我们尝试使用Bellman-Ford算法，并且使用不同的方法对其进行优化。

SPFA

(SPFA)Shortest Path Faster Algorithm)，从Bellman-Ford算法中优化而来。Bellman-Ford算法思路简洁，但是时间复杂度很高。原因是Bellman-Ford算法每轮操作都需要操作每条边，其中有大量无意义的操作。注意到只有当某个顶点的d[u]值改变时，从它出发的边的邻接点v的d[v]值才会改变，由此可以这样优化：使用队列，每次将队首顶点u取出，将从u出发的所有边进行松弛操作，最小化d[v]。若此时顶点v不在队列中就将v入队。如此操作直到队列为空（图中不存在从源点可达的负环），或者某个顶点入队次数超过V-1（图中存在从源点可达的负环）。代码如下

双端队列+SLF优化+LLL优化SPFA

SLF(Small Label First)优化，当队列加入一个新顶点v时，若d[v]比队首d[Q.fornt()]小则加入队首，否则加入队尾。先拓展最小结点可以使程序尽早结束。

LLL(Large Label Last)优化：设队首元素为k，队列中所有d[i]平均值为x，每次松弛操作时判断若d[k]>x，则将k插入队尾，查找下一元素直到找到某一个k使得d[k]<=x，则将k出队进行松弛操作。

一般来说Deque+SLF+LLL可以优化50%左右。
